## Key Features

Query components enable:

- **Flexible Search**: Multiple search patterns from exact matches to semantic similarity
- **Natural Language**: Process queries written in natural language
- **Dynamic Parameters**: Runtime parameter binding and evaluation
- **Advanced Filtering**: Combine multiple filter types for precise results
- **Result Control**: Comprehensive control over result ranking and formatting

<Tip>
The Query module supports both programmatic and natural language queries, making it accessible to both developers and end-users while maintaining powerful customization capabilities.
</Tip>

## Usage Patterns

Common query patterns include:

1. **Semantic Search**: Find content similar in meaning to a query
2. **Filtered Search**: Combine similarity with exact constraints
3. **Natural Language Queries**: Process user queries in plain English
4. **Parameterized Queries**: Use dynamic parameters for reusable query templates
